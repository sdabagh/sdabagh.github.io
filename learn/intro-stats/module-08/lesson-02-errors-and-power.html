2. Type I Error (Œ±)

üìñ Definition: Type I Error

A Type I error occurs when we reject a true null hypothesis. This is also called a false positive.

The probability of making a Type I error is denoted by Œ± (alpha), which is the significance level we choose.

üìù Example 1: Type I Error in Medicine

Medical test scenario:

  • H‚ÇÄ: Patient does not have the disease
  • H‚Çê: Patient has the disease

Type I Error: The test says the patient HAS the disease (reject H‚ÇÄ), but the patient is actually healthy (H‚ÇÄ was true).

Consequence: Unnecessary treatment, anxiety, additional testing, and medical costs for a healthy person.

üìù Example 2: Type I Error in Criminal Justice

  • H‚ÇÄ: Defendant is innocent
  • H‚Çê: Defendant is guilty

Type I Error: Convict an innocent person (reject H‚ÇÄ when it's true).

Consequence: An innocent person goes to jail‚Äîa very serious error!

This is why the criminal justice system uses a very low Œ± ("beyond reasonable doubt").

Controlling Type I Error:

We directly control the Type I error rate by choosing Œ±. Common choices:

  • Œ± = 0.05: Accept 5% chance of Type I error (standard in most research)
  • Œ± = 0.01: Accept only 1% chance (more conservative, fewer false positives)
  • Œ± = 0.10: Accept 10% chance (less conservative, more false positives)

3. Type II Error (Œ≤)

üìñ Definition: Type II Error

A Type II error occurs when we fail to reject a false null hypothesis. This is also called a false negative.

The probability of making a Type II error is denoted by Œ≤ (beta).

üìù Example 3: Type II Error in Medicine

Medical test scenario:

  • H‚ÇÄ: Patient does not have the disease
  • H‚Çê: Patient has the disease

Type II Error: The test says the patient is healthy (fail to reject H‚ÇÄ), but the patient actually HAS the disease (H‚ÇÄ was false).

Consequence: Disease goes untreated, potentially leading to serious health complications or death.

üìù Example 4: Type II Error in Drug Development

  • H‚ÇÄ: New drug is no better than existing treatment
  • H‚Çê: New drug is better than existing treatment

Type II Error: Conclude the new drug doesn't work (fail to reject H‚ÇÄ), when it actually is effective (H‚ÇÄ is false).

Consequence: An effective treatment is rejected and never becomes available to patients.

4. The Relationship Between Œ± and Œ≤

Type I and Type II errors are inversely related:

The Tradeoff:
  • If you decrease Œ± (reduce Type I error), you increase Œ≤ (increase Type II error)
  • If you increase Œ± (increase Type I error), you decrease Œ≤ (reduce Type II error)

Think of it like adjusting the sensitivity of a test:

  • Very strict test (low Œ±): Fewer false alarms, but might miss real effects
  • Lenient test (high Œ±): Catches more real effects, but also more false alarms

Which Error is Worse?

The answer depends on context and consequences:

Scenario Worse Error Strategy
Criminal trial Type I (convict innocent) Use very low Œ± ("beyond reasonable doubt")
Cancer screening Type II (miss cancer) Accept higher Œ± to catch more cases
Quality control (safety) Type II (miss defect) Strict testing to avoid missing defects
Scientific research Type I (false discovery) Standard Œ± = 0.05 to control false claims

üìù Example 5: Choosing Œ± Based on Consequences

Scenario A: Airport Security

  • H‚ÇÄ: Passenger is not a threat
  • Type I Error: Flag innocent passenger (inconvenience)
  • Type II Error: Miss actual threat (catastrophic)
  • Decision: Use higher Œ± to minimize Type II error (better safe than sorry)

Scenario B: Spam Filter

  • H‚ÇÄ: Email is legitimate
  • Type I Error: Mark legitimate email as spam (might miss important message)
  • Type II Error: Let spam through (minor annoyance)
  • Decision: Use lower Œ± to minimize Type I error (don't want to lose important emails)

5. Statistical Power (1 - Œ≤)

üìñ Definition: Power

Statistical power is the probability of correctly rejecting a false null hypothesis. It's calculated as:

Power = 1 - Œ≤

Power represents the test's ability to detect a real effect when one exists. Higher power is better!

What does power tell us?

  • Power = 0.80 (80%): If there really is an effect, we have an 80% chance of detecting it
  • Power = 0.50 (50%): Only a coin flip's chance of detecting a real effect‚Äînot good!
  • Power = 0.95 (95%): Very high chance of detecting a real effect‚Äîexcellent!
Recommended Power: Most researchers aim for power ‚â• 0.80 (80% or higher). This means accepting up to a 20% chance of a Type II error (Œ≤ = 0.20).

Question 1: A pharmaceutical company tests a new drug. What would be a Type I error and a Type II error in this scenario?

H‚ÇÄ: The drug is not effective
H‚Çê: The drug is effective

Type I Error (Œ±): Conclude the drug is effective (reject H‚ÇÄ) when it actually doesn't work (H‚ÇÄ true).

Consequence: Ineffective drug goes to market, patients pay for treatment that doesn't help.

Type II Error (Œ≤): Conclude the drug doesn't work (fail to reject H‚ÇÄ) when it actually is effective (H‚ÇÄ false).

Consequence: Effective treatment never reaches patients who could benefit.

Question 2: A study has Œ± = 0.05 and power = 0.75. What are the probabilities of Type I error, Type II error, and correctly detecting a real effect?

Type I Error (Œ±) = 0.05 (5%)

Type II Error (Œ≤) = 1 - Power = 1 - 0.75 = 0.25 (25%)

Correctly detecting real effect (Power) = 0.75 (75%)

This means if the null hypothesis is false (there really is an effect), we have a 75% chance of correctly rejecting it, and a 25% chance of missing it.

Question 3: A researcher wants to increase the power of their study from 0.70 to 0.85. What are three ways they could do this?

Three ways to increase power:

  1. Increase sample size: Collect data from more participants (most common approach)
  2. Increase Œ±: Use Œ± = 0.10 instead of 0.05 (though this increases Type I error risk)
  3. Reduce variability: Use more precise measurement tools or more homogeneous sample

Best approach: Usually increasing sample size, as it doesn't involve the tradeoffs of the other methods.

  • Type I Error (Œ±): Rejecting a true null hypothesis (false positive)
  • Type II Error (Œ≤): Failing to reject a false null hypothesis (false negative)
  • Œ± and Œ≤ are inversely related‚Äîdecreasing one increases the other
  • Power = 1 - Œ≤: The probability of correctly detecting a real effect
  • We aim for power ‚â• 0.80 (at least 80%)
  • The most practical way to increase power is to increase sample size
  • Which error is worse depends on the real-world consequences of each type
