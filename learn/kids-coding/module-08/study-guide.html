Think Like a Coder: Complete Course Study Guide

A review of all 8 modules, key vocabulary, and skills learned

This study guide covers the entire Think Like a Coder course. Use it to review what you learned, look up vocabulary words, or remember key ideas from any module. You can print it and keep it as a reference!

All 8 Modules at a Glance

Module 1: What is Thinking Like a Coder?

Big Idea: Computers follow exact instructions, step by step. Coding is really about thinking clearly and giving precise instructions.

  • Computers do exactly what you tell them -- nothing more, nothing less
  • The Robot Game showed that vague instructions lead to unexpected results
  • Scratch is a visual coding tool where you snap blocks together
  • Thinking like a coder means being clear, precise, and logical

Module 2: Patterns Everywhere

Big Idea: Patterns are things that repeat or follow a rule. Spotting patterns helps you predict what comes next and find shortcuts.

  • Patterns appear in shapes, numbers, colors, music, and nature
  • Once you spot a pattern, you can predict what comes next
  • Patterns help coders write smarter, shorter code
  • In Scratch, you can create patterns using repeat blocks

Module 3: Step-by-Step Thinking

Big Idea: Decomposition means breaking a big problem into smaller, easier steps. A sequence is a set of instructions in a specific order.

  • Big problems become manageable when you break them into small pieces
  • The order of steps matters -- changing the order changes the result
  • An algorithm is a step-by-step set of instructions to solve a problem
  • In Scratch, sequences are blocks connected from top to bottom

Module 4: Making Decisions

Big Idea: Conditionals let you make decisions based on whether something is true or false. "IF this is true, THEN do that."

  • If/then thinking is how computers (and people) make choices
  • A condition is something that is either true or false
  • Decision trees help you map out all possible choices
  • In Scratch, the "if/then" and "if/then/else" blocks let sprites make decisions

Module 5: Loops and Repetition

Big Idea: Loops let you repeat actions without writing the same instructions over and over. They save time and make code shorter.

  • Repetition is everywhere -- in music, art, daily routines, and nature
  • A loop repeats a set of instructions a certain number of times (or forever)
  • Loops make code shorter and easier to read
  • In Scratch, "repeat" and "forever" blocks create loops

Module 6: Debugging -- Finding and Fixing Mistakes

Big Idea: Mistakes are clues, not failures. Debugging means finding and fixing errors in a calm, systematic way.

  • Every programmer makes mistakes -- even experts
  • A bug is a mistake in code that makes it do something unexpected
  • Debugging means finding the bug and fixing it
  • Strategies: read code carefully, test one thing at a time, check your assumptions
  • A growth mindset means believing you can get better with practice

Module 7: Building Something Real

Big Idea: Design thinking means planning before you build. Good creators ask WHO, WHAT, and HOW, then plan, build, test, and improve.

  • Design thinking: Think, Plan, Build, Test, Improve
  • A storyboard is a set of drawings showing your project scene by scene
  • Build and test in small steps -- do not try to do everything at once
  • Iteration means making something better each time you work on it
  • Kind feedback tells what is good AND suggests improvements

Module 8: Thinking Beyond Code

Big Idea: The thinking skills from coding help with everything -- writing, math, science, games, and everyday life.

  • Logic puzzles use the same skills as coding: step-by-step reasoning, testing, and elimination
  • Strategy games teach you to think ahead and plan moves
  • Decomposition helps with writing; patterns help with math; debugging helps with fixing mistakes
  • Thinking like a coder is a lifelong skill, not just a computer skill

Key Skills You Learned

Spot patterns Notice when things repeat or follow rules, and use patterns to predict what comes next
Break problems into steps Take a big, scary problem and turn it into a list of small, doable tasks
Make decisions with logic Use if/then thinking to choose the right action based on what is true
