Key insight to land: Consistency is everything. If you change how you record answers halfway through, your data is ruined. Establish the recording method BEFORE starting.
📦 Materials Needed
Clipboards (1 per student) Pencils Tally recording sheets (from worksheet) Rulers (for measurement station) 3–4 objects to measure per group Timer (phone or classroom clock)
💡 The mini-survey works best if students can move around the classroom. Clear a walking path if possible.
📚 Key Vocabulary
Survey — asking people questions to collect data
Observation — watching and recording what you see
Measurement — using tools to get exact numbers
Tally mark — a line (| | | | ̶|) used for fast counting
Consistency — recording answers the same way every time
Data collection error — a mistake that makes data unreliable
💬 Discussion Questions + Teacher Notes
  • "What are three ways we can collect data?"
    → Survey (ask people), observation (watch and record), measurement (use tools). Give one example of each before asking students.
  • "Why use tally marks instead of writing 1, 2, 3…?"
    → Tally marks are faster when you're listening and recording at the same time. The grouped-of-5 system makes counting easy at the end.
  • "What could go wrong while you're surveying people?"
    → Brainstorm errors: forgetting to record, misunderstanding the answer, changing the question, tallying the wrong column. These become the Error Detective scenarios.
  • "Is asking someone twice a problem?"
    → Yes! Each person should only be counted once. Double-counting inflates one answer.
  • "Why does consistency matter?"
    → If you record "yes," then "Y," then a checkmark, you can't compare them later. Same format every time = usable data.
🎮 Mini-Survey Activity Setup
Students use their data question from Session 2 to survey 10 classmates. They record results using tally marks on their worksheet.
Tally mark teaching moment — draw on board:
|   | |   | | |   | | | |   ||||̶ = 1, 2, 3, 4, 5
The 5th mark CROSSES the previous 4. Groups of 5 make counting fast!
Survey rules to post on board:
  1. Ask the question exactly as written — no changes
  2. Give the answer choices — don't suggest one
  3. Record with a tally mark immediately
  4. Each person only gets asked once
  5. Say "thank you" after each response!
💡 Walk around while students survey. Listen for leading language and gently redirect.
🔍 Error Detective Scenarios
Present these 3 scenarios. Groups identify the error.
Scenario 1: Mia asks "Do you prefer cats, dogs, or fish?" but records the first 5 answers as words and the next 5 as tally marks. Error: inconsistent recording method.
Scenario 2: Sam asks his question but keeps adding "...most people say pizza, just so you know." Error: leading the respondents.
Scenario 3: Dev surveys 10 people but accidentally asks his best friend twice. Error: double-counting one person.
✍️ Wrap-Up Prompt
Write on board:
"What was the hardest part of collecting data today? What would you do differently next time?"
5 min quiet reflection → 2 share aloud. Bridge to Session 4: "Now all that data needs to be organized!"
🧠 ND-Friendly Tips
  • Social anxiety — Approaching strangers (even classmates) is hard for some kids. Allow them to survey only students they're comfortable with, or survey in pairs.
  • Demonstrate first — Model the full survey interaction with a student volunteer before releasing the class.
  • Tally marks — Some students need extra practice. Let them count on fingers first, then transfer to tally.
  • Timer visible — Show how long the survey window is. "You have 12 minutes to ask 10 people."
  • Error detective — Frame errors as learning, not failure. "Even professional data scientists make these mistakes!"
